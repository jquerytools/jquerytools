Input validation logic

The wizard contains a simple validation logic that only checks that the required fields are not blank. It checks for input fields that are nested inside an li.required element and makes sure that they have a value. White space characters are not accepted. There are hundreds of different ways to implement input validation. Some rely solely on client-side validation with JavaScript and some use only server-side validation. Many times both are used simultaneously. Because of the wild world of input validation there is no full-blown solution presented here.

I have added lot's of comments so that you'll understand what's happening. Notice that the "drawer" logic is a very simple one-liner although the effect is very useful.
